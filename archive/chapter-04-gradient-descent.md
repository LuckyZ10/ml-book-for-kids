# 第四章 一步一步变得更好——梯度下降的直觉

## 4.0 引子：迷雾中的下山者

想象一下，你是一个登山者，名叫小明。你在一个浓雾弥漫的早晨被困在了一座高山的山顶。能见度极低，你几乎看不清前方三米之外的东西。手机没有信号，GPS也无法使用。你只知道一件事：必须尽快下山，回到安全的山谷。

你会怎么做？

小明想了想，蹲下来，用手触摸地面。他感觉到脚下的坡度——左侧微微向上倾斜，右侧则向下倾斜。于是，他小心翼翼地朝右侧迈出了一小步。然后，他再次蹲下，重新感受新的位置上的坡度，再朝下坡的方向迈出一步。

一步，又一步。小明就这样摸索着前进。有时候，他会遇到平缓的坡地，需要走很多步才能有明显的下降；有时候，他会遇到陡峭的悬崖边缘，必须格外小心，控制步伐的大小，以免失足跌落。

这个故事，就是**梯度下降**（Gradient Descent）最生动的比喻。在机器学习中，我们就是那个在浓雾中下山的小明，而损失函数（Loss Function）的表面就是那座山。我们的目标是找到这座山的最低点——也就是损失函数的最小值。

## 4.1 直观理解梯度下降

### 4.1.1 从几何直观说起

让我们暂时离开小明的山，来到数学的世界。

想象你站在一个山坡上的某一点。如果你想以最快的速度下山，你应该朝哪个方向走？答案是：**最陡峭的下降方向**。

这个方向，数学上称为**负梯度方向**（Negative Gradient Direction）。

梯度（Gradient）是一个向量，它有两个重要性质：
1. **方向**：指向函数增长最快的方向
2. **大小**：表示增长的速率

所以，如果我们想最小化一个函数，我们就应该朝梯度的反方向移动——就像水总是从高处流向低处，沿着最陡峭的坡度下降。

### 4.1.2 梯度下降的基本思想

梯度下降算法的核心思想可以用一句话概括：

> **在当前位置计算梯度，然后朝梯度的反方向迈出一小步，重复这个过程直到收敛。**

用数学语言表达，如果我们想最小化函数 $f(\mathbf{x})$，其中 $\mathbf{x} = (x_1, x_2, \ldots, x_n)$ 是参数向量，那么更新规则是：

$$\mathbf{x}_{t+1} = \mathbf{x}_t - \eta \nabla f(\mathbf{x}_t)$$

其中：
- $\mathbf{x}_t$ 是第 $t$ 步时的参数值
- $\eta$ 是学习率（Learning Rate），控制每一步的大小
- $\nabla f(\mathbf{x}_t)$ 是函数在 $\mathbf{x}_t$ 处的梯度

## 4.2 数学推导：从零开始的完整推导

### 4.2.1 单变量函数的梯度下降

让我们从最简单的情况开始：单变量函数 $f(x)$。

**泰勒展开**是理解梯度下降的关键。回忆一下，函数 $f(x)$ 在点 $x_0$ 附近的一阶泰勒展开为：

$$f(x) \approx f(x_0) + f'(x_0)(x - x_0)$$

假设我们从 $x_t$ 出发，想要移动到 $x_{t+1} = x_t + \Delta x$，使得 $f(x_{t+1}) < f(x_t)$。

根据泰勒展开：
$$f(x_{t+1}) \approx f(x_t) + f'(x_t) \cdot \Delta x$$

我们希望 $f(x_{t+1}) < f(x_t)$，即：
$$f'(x_t) \cdot \Delta x < 0$$

这意味着 $\Delta x$ 应该与 $f'(x_t)$ 符号相反。最简单的选择是：
$$\Delta x = -\eta \cdot f'(x_t)$$

其中 $\eta > 0$ 是一个小常数，称为**学习率**。

因此，我们得到单变量梯度下降的更新公式：
$$x_{t+1} = x_t - \eta \cdot f'(x_t)$$

### 4.2.2 多变量函数的梯度下降

现在扩展到多变量情况。设 $f: \mathbb{R}^n \rightarrow \mathbb{R}$ 是一个多元函数，$\mathbf{x} = (x_1, x_2, \ldots, x_n)^T$。

**梯度**定义为所有偏导数组成的向量：
$$\nabla f(\mathbf{x}) = \begin{bmatrix} \frac{\partial f}{\partial x_1} \\ \frac{\partial f}{\partial x_2} \\ \vdots \\ \frac{\partial f}{\partial x_n} \end{bmatrix}$$

多变量函数的一阶泰勒展开为：
$$f(\mathbf{x} + \Delta \mathbf{x}) \approx f(\mathbf{x}) + \nabla f(\mathbf{x})^T \Delta \mathbf{x}$$

我们希望 $f(\mathbf{x} + \Delta \mathbf{x}) < f(\mathbf{x})$，即：
$$\nabla f(\mathbf{x})^T \Delta \mathbf{x} < 0$$

根据柯西-施瓦茨不等式，当 $\Delta \mathbf{x}$ 与 $-\nabla f(\mathbf{x})$ 同方向时，内积 $\nabla f(\mathbf{x})^T \Delta \mathbf{x}$ 达到最小（最负）。因此，最优的下降方向就是负梯度方向。

取 $\Delta \mathbf{x} = -\eta \nabla f(\mathbf{x})$，得到多变量梯度下降的更新公式：
$$\mathbf{x}_{t+1} = \mathbf{x}_t - \eta \nabla f(\mathbf{x}_t)$$

### 4.2.3 收敛性分析

在什么条件下梯度下降会收敛到最优解？

**定理（梯度下降的收敛性）**：假设函数 $f$ 是 $L$-光滑（L-smooth）且 $\mu$-强凸（strongly convex）的，即：
- $L$-光滑：$\|\nabla f(\mathbf{x}) - \nabla f(\mathbf{y})\| \leq L \|\mathbf{x} - \mathbf{y}\|$
- $\mu$-强凸：$f(\mathbf{y}) \geq f(\mathbf{x}) + \nabla f(\mathbf{x})^T(\mathbf{y}-\mathbf{x}) + \frac{\mu}{2}\|\mathbf{y}-\mathbf{x}\|^2$

如果选择学习率 $\eta \leq \frac{1}{L}$，那么梯度下降满足：
$$f(\mathbf{x}_t) - f(\mathbf{x}^*) \leq \left(1 - \frac{\mu}{L}\right)^t (f(\mathbf{x}_0) - f(\mathbf{x}^*))$$

其中 $\mathbf{x}^*$ 是最优点，$\kappa = L/\mu$ 称为**条件数**（Condition Number）。

这个定理告诉我们：
1. 梯度下降的收敛速度是**线性**（几何级数）的
2. 条件数越大（函数"扁平"），收敛越慢
3. 学习率不能超过 $1/L$，否则可能发散

## 4.3 学习率的选择：艺术还是科学？

### 4.3.1 学习率的重要性

学习率 $\eta$ 是梯度下降中最重要的超参数，它决定了每一步迈出的"步子"有多大。

**学习率太小**：
- 优点：稳定，不容易错过最优解
- 缺点：收敛极慢，需要更多的迭代次数

**学习率太大**：
- 优点：初期收敛快
- 缺点：可能震荡，甚至发散

### 4.3.2 可视化理解

让我们用一个简单的二次函数 $f(x) = x^2$ 来可视化不同学习率的效果。

从 $x_0 = 5$ 开始，梯度为 $f'(x) = 2x$。

**情况1：学习率适中 ($\eta = 0.1$)**
```
迭代0: x = 5.00, f(x) = 25.00
迭代1: x = 5 - 0.1×10 = 4.00, f(x) = 16.00
迭代2: x = 4 - 0.1×8 = 3.20, f(x) = 10.24
迭代3: x = 3.2 - 0.1×6.4 = 2.56, f(x) = 6.55
...
```
稳定收敛！

**情况2：学习率太大 ($\eta = 0.8$)**
```
迭代0: x = 5.00, f(x) = 25.00
迭代1: x = 5 - 0.8×10 = -3.00, f(x) = 9.00
迭代2: x = -3 - 0.8×(-6) = 1.80, f(x) = 3.24
迭代3: x = 1.8 - 0.8×3.6 = -1.08, f(x) = 1.17
...
```
震荡，但最终可能收敛。

**情况3：学习率过大 ($\eta = 1.1$)**
```
迭代0: x = 5.00, f(x) = 25.00
迭代1: x = 5 - 1.1×10 = -6.00, f(x) = 36.00  ← 更差了！
迭代2: x = -6 - 1.1×(-12) = 7.20, f(x) = 51.84
迭代3: x = 7.2 - 1.1×14.4 = -8.64, f(x) = 74.65
...
```
发散！

### 4.3.3 学习率的选择策略

在实践中，有几种常用的学习率选择策略：

**1. 固定学习率**
最简单的方法，选择一个常数学习率（如 0.01, 0.001）。

**2. 学习率衰减**
随着训练进行，逐渐减小学习率：
$$\eta_t = \frac{\eta_0}{1 + \gamma t}$$
或
$$\eta_t = \eta_0 \cdot \gamma^t$$

**3. 自适应学习率**
根据梯度历史自动调整学习率（后面会详细介绍 AdaGrad, Adam 等方法）。

**4. 学习率调度**
- **步进衰减**：每隔一定轮数，学习率乘以一个小于1的因子
- **指数衰减**：学习率按指数函数衰减
- **余弦退火**：学习率按余弦函数周期性变化

## 4.4 鞍点与局部最优：优化路上的陷阱

### 4.4.1 临界点类型

在优化过程中，我们会遇到梯度为零的点，称为**临界点**（Critical Points）。临界点有三种类型：

**1. 局部最小值（Local Minimum）**
在该点的邻域内，函数值最小。Hessian矩阵正定（所有特征值 > 0）。

**2. 局部最大值（Local Maximum）**
在该点的邻域内，函数值最大。Hessian矩阵负定（所有特征值 < 0）。

**3. 鞍点（Saddle Point）**
在某些方向上是局部最小值，在另一些方向上是局部最大值。Hessian矩阵既有正特征值，也有负特征值。

### 4.4.2 鞍点的困扰

在深度学习的高维优化问题中，鞍点比局部最小值更加普遍。研究表明，在随机高维非凸函数中，局部最小值非常罕见，而鞍点却无处不在。

**为什么鞍点是问题？**

在鞍点处，梯度为零，但某些方向上函数值可以进一步降低。纯梯度下降会在鞍点附近"卡住"或移动缓慢，因为这些区域的梯度很小。

### 4.4.3 逃离鞍点的策略

**1. 随机扰动**
Ge等人（2015）证明，添加随机噪声的梯度下降可以在多项式时间内逃离鞍点：
$$\mathbf{x}_{t+1} = \mathbf{x}_t - \eta \nabla f(\mathbf{x}_t) + \epsilon_t$$

其中 $\epsilon_t$ 是随机噪声。

**2. 动量方法**
动量方法可以帮助算法"冲过"平坦区域和鞍点（将在4.6节详细介绍）。

**3. 二阶信息**
使用Hessian矩阵的特征值信息可以检测和逃离鞍点，但计算成本高。

**4. 随机梯度下降（SGD）**
有趣的是，SGD中的随机噪声实际上有助于逃离鞍点。这是因为每次迭代的梯度估计有噪声，提供了逃离鞍点所需的"扰动"。

### 4.4.4 局部最优真的那么可怕吗？

对于深度神经网络，研究发现：
- 不同局部最优的函数值往往非常接近
- 宽的（平坦的）局部最小值通常具有更好的泛化能力
- 在高维空间中，几乎所有局部最小值都是近似全局最小值

因此，找到一个"足够好"的局部最小值往往比找到全局最小值更重要。

## 4.5 随机梯度下降：大数据时代的救星

### 4.5.1 从批量梯度下降到随机梯度下降

在机器学习中，我们通常需要最小化训练集上的平均损失：
$$f(\mathbf{x}) = \frac{1}{N} \sum_{i=1}^{N} f_i(\mathbf{x})$$

其中 $N$ 是训练样本数量，$f_i(\mathbf{x})$ 是第 $i$ 个样本的损失。

**批量梯度下降（Batch Gradient Descent, BGD）**：
每次迭代计算所有样本的梯度：
$$\nabla f(\mathbf{x}) = \frac{1}{N} \sum_{i=1}^{N} \nabla f_i(\mathbf{x})$$

**问题**：当 $N$ 很大时（如百万、十亿级样本），计算完整梯度非常昂贵！

### 4.5.2 随机梯度下降（Stochastic Gradient Descent, SGD）

1951年，Herbert Robbins 和 Sutton Monro 提出了**随机逼近**（Stochastic Approximation）算法，奠定了SGD的理论基础。1998年，Léon Bottou 将SGD引入机器学习领域。

SGD的核心思想：每次只用一个样本（或一小批样本）来估计梯度！

**算法流程**：
1. 随机打乱训练数据
2. 对于每个样本 $i$（或小批量）：
   $$\mathbf{x}_{t+1} = \mathbf{x}_t - \eta_t \nabla f_i(\mathbf{x}_t)$$

### 4.5.3 SGD的优势

**1. 计算效率高**
每次迭代只需要处理一个样本，计算复杂度从 $O(N)$ 降低到 $O(1)$。

**2. 内存友好**
不需要一次性加载所有数据到内存。

**3. 在线学习**
可以处理流式数据，样本到来时立即更新模型。

**4. 有助于逃离局部最优和鞍点**
随机噪声提供了探索的能力。

### 4.5.4 SGD的挑战与解决

**挑战1：梯度方差大**
单个样本的梯度是真实梯度的有噪声估计。

**解决方案**：
- **小批量梯度下降**（Mini-batch GD）：使用 $m$ 个样本（如 $m=32, 64, 128$）估计梯度，平衡了计算效率和估计准确性。
- **动量方法**：累积历史梯度，平滑更新方向。

**挑战2：学习率选择困难**
SGD对学习率更敏感。

**解决方案**：
- 学习率衰减
- 自适应学习率方法（AdaGrad, RMSProp, Adam）

**SGD的收敛性**：
对于凸函数，如果使用满足以下条件的学习率：
$$\sum_{t=1}^{\infty} \eta_t = \infty, \quad \sum_{t=1}^{\infty} \eta_t^2 < \infty$$

SGD几乎必然收敛到最优解。典型的选择是 $\eta_t = \frac{\eta_0}{t}$ 或 $\eta_t = \frac{\eta_0}{\sqrt{t}}$。

## 4.6 动量法：物理直觉的启发

### 4.6.1 从物理世界获得灵感

想象一个小球从山坡上滚下来。它不会每一步都停下来重新计算方向，而是依靠**惯性**继续滚动。即使遇到小的障碍物或凹陷，它也能凭借惯性越过。

这就是**动量方法**（Momentum Method）的物理直觉。

### 4.6.2 Polyak动量（重球法）

1964年，Boris Polyak 提出了经典的动量方法，也称为**重球法**（Heavy Ball Method）。

**更新公式**：
$$\mathbf{v}_{t+1} = \gamma \mathbf{v}_t + \eta \nabla f(\mathbf{x}_t)$$
$$\mathbf{x}_{t+1} = \mathbf{x}_t - \mathbf{v}_{t+1}$$

其中：
- $\mathbf{v}_t$ 是速度（动量）向量
- $\gamma \in [0, 1)$ 是动量系数（通常设为 0.9）
- $\eta$ 是学习率

也可以写成等价的形式：
$$\mathbf{x}_{t+1} = \mathbf{x}_t - \eta \nabla f(\mathbf{x}_t) + \gamma (\mathbf{x}_t - \mathbf{x}_{t-1})$$

这表明当前更新不仅基于当前梯度，还考虑了上一步的更新方向。

### 4.6.3 Nesterov加速梯度

1983年，Yurii Nesterov 提出了一种改进的动量方法，称为**Nesterov加速梯度**（Nesterov Accelerated Gradient, NAG）。

**直觉**：与其在"当前位置"计算梯度，不如在"预计到达的位置"计算梯度。

**更新公式**：
$$\mathbf{v}_{t+1} = \gamma \mathbf{v}_t + \eta \nabla f(\mathbf{x}_t - \gamma \mathbf{v}_t)$$
$$\mathbf{x}_{t+1} = \mathbf{x}_t - \mathbf{v}_{t+1}$$

或者等价地：
$$\mathbf{y}_t = \mathbf{x}_t + \gamma (\mathbf{x}_t - \mathbf{x}_{t-1})$$
$$\mathbf{x}_{t+1} = \mathbf{y}_t - \eta \nabla f(\mathbf{y}_t)$$

### 4.6.4 动量方法的优势

**1. 加速收敛**
在峡谷形的损失表面（某些方向很陡，某些方向很平缓），动量方法可以沿陡坡快速下降，同时减少在平缓方向的震荡。

**2. 逃离局部最优和鞍点**
累积的动量可以帮助算法"冲过"小的局部最优或鞍点。

**3. 平滑更新**
动量平均了历史梯度，减少了随机梯度的方差。

**4. 处理病态条件**
对于条件数很大的问题，动量方法比纯梯度下降收敛快得多。

### 4.6.5 动量系数的选择

- $\gamma = 0$：退化为普通梯度下降
- $\gamma = 0.9$：最常用的默认值
- $\gamma$ 过大（如 >0.99）：可能导致震荡或发散

## 4.7 现代优化器：自适应学习率的崛起

### 4.7.1 自适应学习率的动机

在标准SGD中，所有参数使用相同的学习率。但在深度神经网络中，不同层、不同参数的梯度大小可能差异很大：
- 某些参数需要大幅更新
- 某些参数只需要微调

**问题**：如何为每个参数自动选择合适的学习率？

### 4.7.2 AdaGrad：累积历史梯度

2011年，John Duchi 等人提出了 **AdaGrad**（Adaptive Gradient）。

**核心思想**：为每个参数维护一个累积平方梯度的历史，用这个历史来缩放学习率。

**更新公式**：
$$\mathbf{g}_t = \nabla f(\mathbf{x}_t)$$
$$\mathbf{r}_t = \mathbf{r}_{t-1} + \mathbf{g}_t \odot \mathbf{g}_t$$
$$\mathbf{x}_{t+1} = \mathbf{x}_t - \frac{\eta}{\sqrt{\mathbf{r}_t} + \epsilon} \odot \mathbf{g}_t$$

其中：
- $\odot$ 表示逐元素乘法
- $\mathbf{r}_t$ 累积了历史梯度的平方
- $\epsilon$ 是防止除零的小常数（如 $10^{-8}$）

**特点**：
- 对于频繁出现的特征（梯度大），学习率减小
- 对于稀疏特征（梯度小），学习率增大
- 非常适合稀疏数据（如自然语言处理）

**缺点**：
- 累积梯度平方会导致学习率单调递减，最终可能变得太小

### 4.7.3 RMSProp：指数移动平均

2012年，Geoffrey Hinton 在 Coursera 课程中提出了 **RMSProp**（Root Mean Square Propagation），作为对AdaGrad的改进。

**核心思想**：使用指数移动平均代替累积和，让历史梯度"遗忘"旧信息。

**更新公式**：
$$\mathbf{g}_t = \nabla f(\mathbf{x}_t)$$
$$\mathbf{r}_t = \rho \mathbf{r}_{t-1} + (1-\rho) \mathbf{g}_t \odot \mathbf{g}_t$$
$$\mathbf{x}_{t+1} = \mathbf{x}_t - \frac{\eta}{\sqrt{\mathbf{r}_t} + \epsilon} \odot \mathbf{g}_t$$

其中 $\rho$ 是衰减率（通常设为 0.9）。

**特点**：
- 解决了AdaGrad学习率单调递减的问题
- 在非凸优化中表现更好
- 适合循环神经网络（RNN）

### 4.7.4 Adam：动量 + 自适应学习率

2014年，Diederik Kingma 和 Jimmy Ba 提出了 **Adam**（Adaptive Moment Estimation），结合了动量法和自适应学习率的优点。

**核心思想**：同时估计梯度的一阶矩（均值）和二阶矩（未中心化的方差）。

**更新公式**：
$$\mathbf{g}_t = \nabla f(\mathbf{x}_t)$$
$$\mathbf{m}_t = \beta_1 \mathbf{m}_{t-1} + (1-\beta_1) \mathbf{g}_t$$
$$\mathbf{v}_t = \beta_2 \mathbf{v}_{t-1} + (1-\beta_2) \mathbf{g}_t \odot \mathbf{g}_t$$

**偏差修正**（因为初始值为0，前几步估计有偏差）：
$$\hat{\mathbf{m}}_t = \frac{\mathbf{m}_t}{1-\beta_1^t}, \quad \hat{\mathbf{v}}_t = \frac{\mathbf{v}_t}{1-\beta_2^t}$$

**参数更新**：
$$\mathbf{x}_{t+1} = \mathbf{x}_t - \frac{\eta}{\sqrt{\hat{\mathbf{v}}_t} + \epsilon} \hat{\mathbf{m}}_t$$

**默认参数**：
- $\beta_1 = 0.9$（一阶矩衰减率）
- $\beta_2 = 0.999$（二阶矩衰减率）
- $\eta = 0.001$
- $\epsilon = 10^{-8}$

**Adam的优势**：
1. 计算高效，内存需求小
2. 适合大规模数据和高维参数空间
3. 对学习率不太敏感（默认0.001在大多数情况下都有效）
4. 自动调整学习率，无需手动调参

### 4.7.5 优化器选择指南

| 优化器 | 优点 | 缺点 | 适用场景 |
|--------|------|------|----------|
| SGD + Momentum | 泛化性好，收敛到更平坦的局部最小值 | 需要调学习率 | 计算机视觉、需要最好泛化性能 |
| AdaGrad | 适合稀疏数据 | 学习率单调递减 | 自然语言处理（已较少使用） |
| RMSProp | 适合非凸问题 | 需要调衰减率 | RNN、非平稳目标 |
| Adam | 易用、收敛快 | 可能泛化略差于SGD | 大多数深度学习任务、默认选择 |

## 4.8 本章总结

从柯西1847年的原创论文，到Robbins和Monro1951年的随机逼近，再到今天深度学习中广泛使用的Adam优化器，梯度下降算法已经走过了近180年的发展历程。

本章我们学习了：

1. **梯度下降的直观理解**：像下山一样，沿着最陡峭的方向逐步下降
2. **数学推导**：基于泰勒展开，负梯度方向是局部下降最快的方向
3. **学习率选择**：太小收敛慢，太大可能发散，需要精心选择
4. **鞍点与局部最优**：高维优化中鞍点更常见，随机性有助于逃离
5. **随机梯度下降**：用随机梯度估计替代完整梯度，大大提高计算效率
6. **动量方法**：借鉴物理直觉，加速收敛并帮助逃离局部最优
7. **自适应优化器**：自动调整学习率，让优化更加智能

梯度下降是现代机器学习的基石。理解它的原理、优缺点和各种变体，对于训练深度学习模型至关重要。

记住小明下山的故事：在浓雾中，每一步都基于脚下的坡度，一步一步，终将到达山谷。

## 练习题

### 练习题1：基础概念
**问题**：解释为什么梯度下降算法中，参数更新方向是负梯度方向。用单变量函数 $f(x) = x^2$ 举例说明。

**解答**：
在单变量函数中，导数 $f'(x)$ 表示函数在该点的变化率：
- 当 $f'(x) > 0$ 时，函数向右（$x$ 增大方向）增加
- 当 $f'(x) < 0$ 时，函数向左（$x$ 减小方向）增加

要使函数值减小，我们应该朝导数的反方向移动：
- 当 $f'(x) > 0$ 时，向左移动（减小 $x$）
- 当 $f'(x) < 0$ 时，向右移动（增大 $x$）

这正是负导数方向 $-f'(x)$。

对于 $f(x) = x^2$，$f'(x) = 2x$：
- 在 $x = 5$ 处，$f'(5) = 10 > 0$，应该向左移动（减小 $x$）
- 更新：$x_{new} = 5 - \eta \cdot 10 = 5 - 10\eta < 5$
- 这样 $f(x_{new}) < f(5)$，函数值减小。

---

### 练习题2：学习率选择
**问题**：考虑函数 $f(x) = x^2$，初始点 $x_0 = 10$。如果使用学习率 $\eta = 0.1$，需要多少步才能使 $x_t < 0.01$？如果 $\eta = 1.1$ 会发生什么？

**解答**：

**情况1：$\eta = 0.1$**

更新公式：$x_{t+1} = x_t - 0.1 \cdot 2x_t = x_t - 0.2x_t = 0.8x_t$

这是一个等比数列：$x_t = 10 \cdot (0.8)^t$

我们需要 $10 \cdot (0.8)^t < 0.01$，即 $(0.8)^t < 0.001$

取对数：$t \cdot \ln(0.8) < \ln(0.001) = -6.91$

$t > \frac{-6.91}{-0.223} \approx 31$

所以需要约31步。

**情况2：$\eta = 1.1$**

更新公式：$x_{t+1} = x_t - 1.1 \cdot 2x_t = x_t - 2.2x_t = -1.2x_t$

每一步 $x$ 的绝对值增大1.2倍！
- $x_0 = 10$
- $x_1 = -12$
- $x_2 = 14.4$
- ...

函数值越来越大，算法**发散**。

收敛条件要求 $|1 - 2\eta| < 1$，即 $0 < \eta < 1$。

### 练习题3：动量方法
**问题**：比较普通梯度下降和带动量的梯度下降在"峡谷"形状损失函数上的表现。假设损失函数为 $f(x, y) = 0.1x^2 + y^2$，解释为什么动量方法会更快收敛。

**解答**：

**梯度分析**：
- $\frac{\partial f}{\partial x} = 0.2x$（$x$方向梯度较小）
- $\frac{\partial f}{\partial y} = 2y$（$y$方向梯度较大）

这个函数在$y$方向很陡，在$x$方向很平缓，形成一个"峡谷"。

**普通梯度下降**：
在点 $(10, 1)$ 处：
- $x$方向梯度：$0.2 \times 10 = 2$
- $y$方向梯度：$2 \times 1 = 2$

两个方向的梯度大小相同，更新步长相等。但最优解在 $y=0$ 附近，$x$方向的更新相对实际需要的步长来说太快了（相对梯度大小而言），会导致在$y$方向震荡，在$x$方向进展缓慢。

**动量梯度下降**：
动量累积了历史梯度方向。在峡谷中滚动时：
- 沿峡谷方向的动量持续累积
- 垂直于峡谷方向的震荡相互抵消

这就像一个小球在峡谷中滚动：它会获得沿峡谷方向的速度，而横向的震荡会被抑制，从而快速到达谷底。

### 练习题4：自适应学习率
**问题**：解释AdaGrad和RMSProp的区别。为什么RMSProp通常比AdaGrad更适合训练深度神经网络？

**解答**：

**AdaGrad**：
$$r_t = r_{t-1} + g_t^2$$
累积所有历史梯度的平方和。

**问题**：随着时间推移，$r_t$ 单调递增，导致学习率 $\eta / \sqrt{r_t}$ 单调递减。在深度网络训练后期，学习率可能变得极小，模型停止学习。

**RMSProp**：
$$r_t = \rho r_{t-1} + (1-\rho) g_t^2$$
使用指数移动平均，只保留近期的梯度信息。

**优势**：
1. $r_t$ 不会无限增长，学习率不会单调递减到接近零
2. 能够适应损失函数的非凸形状
3. 在训练后期仍然保持适当的学习率

**类比**：
- AdaGrad像是一个从不遗忘的学生，早期的错误（大梯度）会一直影响后期的学习率
- RMSProp像一个会"遗忘"的学生，更关注近期的表现，因此更灵活

这就是为什么RMSProp在深度神经网络（通常是非凸优化）中表现更好。

## 基础题（3道）

### 练习1：概念理解 ⭐

请用自己的话解释以下概念：

**问题1**：什么是AGNES算法？它和DIANA算法有什么区别？

<details>
<summary>参考答案</summary>

AGNES（Agglomerative Nesting）是自底向上的凝聚层次聚类算法：
- 初始时每个点自成一簇
- 逐步合并距离最近的两个簇
- 直到所有点合并成一个簇或达到指定簇数

DIANA（Divisive Analysis）是自顶向下的分裂层次聚类算法：
- 初始时所有点在一个大簇中
- 逐步将最不相似的点分裂出去
- 直到每个点自成一簇或达到指定簇数

**主要区别**：
- AGNES是自底向上，DIANA是自顶向下
- AGNES更简单常用，DIANA计算复杂度更高
- AGNES适合发现小的簇，DIANA适合发现大的簇

</details>

**问题2**：什么是DBSCAN中的"核心点"和"边界点"？

<details>
<summary>参考答案</summary>

**核心点（Core Point）**：
- 在其ε-邻域内包含至少MinPts个点（包括自己）
- 是簇的"骨架"，能够扩展出整个簇

**边界点（Border Point）**：
- 不是核心点
- 但在某个核心点的ε-邻域内
- 是簇的"边缘成员"

**区别**：
- 核心点可以"拉朋友"扩展簇
- 边界点只能被核心点"拉入"簇
- 边界点不能扩展新的点

</details>

**问题3**：轮廓系数的取值范围是多少？s(i)=0.8表示什么含义？

<details>
<summary>参考答案</summary>

**轮廓系数取值范围**：[-1, 1]

**s(i) = 0.8 的含义**：
- 非常接近1，表示优秀的聚类效果
- 点i与自己的簇非常紧密（a(i)很小）
- 点i与最近的其他簇距离很远（b(i)很大）
- 点i被分到了正确的簇中

**各区间含义**：
- s(i) ≈ 1：完美聚类
- s(i) ≈ 0：点在两个簇的边界上
- s(i) ≈ -1：点可能被分到了错误的簇

</details>

### 练习2：手工计算 ⭐⭐

给定以下3个二维数据点：
- A(0, 0)
- B(3, 4)
- C(6, 0)

**问题1**：使用欧氏距离，计算AB、BC、AC之间的距离。

<details>
<summary>参考答案</summary>

```
欧氏距离公式: d(p1, p2) = √[(x2-x1)² + (y2-y1)²]

d(A, B) = √[(3-0)² + (4-0)²] = √[9 + 16] = √25 = 5
d(B, C) = √[(6-3)² + (0-4)²] = √[9 + 16] = √25 = 5
d(A, C) = √[(6-0)² + (0-0)²] = √36 = 6
```

所以：AB = 5, BC = 5, AC = 6

</details>

**问题2**：如果使用单链接（Single Linkage）聚类，当聚成2个簇时，结果是什么？

<details>
<summary>参考答案</summary>

单链接距离 = 两个簇中最近的两个点之间的距离

初始：{A}, {B}, {C} 三个簇

第一步：找距离最近的两个簇
- d({A}, {B}) = 5
- d({B}, {C}) = 5
- d({A}, {C}) = 6

假设先合并A和B：{A,B}, {C}

第二步：计算新簇{A,B}与{C}的单链接距离
- d({A,B}, {C}) = min(d(A,C), d(B,C)) = min(6, 5) = 5

此时只剩2个簇：{A,B} 和 {C}

**最终聚类结果**：
- 簇1：{A, B}
- 簇2：{C}

</details>

**问题3**：如果使用全链接（Complete Linkage）聚类，当聚成2个簇时，结果是什么？

<details>
<summary>参考答案</summary>

全链接距离 = 两个簇中最远的两个点之间的距离

初始：{A}, {B}, {C} 三个簇

第一步：找距离最近的两个簇
- d({A}, {B}) = 5
- d({B}, {C}) = 5
- d({A}, {C}) = 6

假设先合并A和B：{A,B}, {C}

第二步：计算新簇{A,B}与{C}的全链接距离
- d({A,B}, {C}) = max(d(A,C), d(B,C)) = max(6, 5) = 6

此时只剩2个簇：{A,B} 和 {C}

**最终聚类结果**：
- 簇1：{A, B}
- 簇2：{C}

**注意**：在这个简单例子中，单链接和全链接结果相同，但在复杂数据中往往不同。

</details>

### 练习3：参数选择 ⭐⭐

**问题1**：在DBSCAN中，如果ε设置得太小，会发生什么？

<details>
<summary>参考答案</summary>

**ε太小的后果**：
1. **大部分点变成噪声**：每个点的ε-邻域内都没有足够多点
2. **簇被切得太碎**：原本连续的簇被分割成很多小碎片
3. **发现过多的小簇**：每个小密集区域被视为独立簇
4. **召回率低**：很多应该在一起的点被分开了

**形象比喻**：
就像用太小的渔网捕鱼，大鱼也网不住，只能抓到一些零星的小鱼。

</details>

**问题2**：如果MinPts设置得太大，会发生什么？

<details>
<summary>参考答案</summary>

**MinPts太大的后果**：
1. **小簇被误判为噪声**：小密集区域无法达到MinPts要求
2. **边界点增多**：更多核心点变成边界点
3. **簇的核心区域变小**：簇的内部"骨架"变小
4. **对噪声更敏感**：需要更多点才能确认一个核心区域

**形象比喻**：
就像要求"必须有10个朋友才算社交达人"，结果大部分人都成了"孤独者"。

</details>

**问题3**：如何用量化的方法选择合适的ε值？

<details>
<summary>参考答案</summary>

**K-距离图法（K-distance Graph）**：

步骤：
1. 对每个点，计算它到第K个最近邻居的距离（K = MinPts - 1）
2. 将所有K-距离按降序排列
3. 绘制K-距离曲线
4. 找到曲线的"肘部"（elbow point）
5. 肘部对应的K-距离即为最佳ε值

**原理**：
- 核心点的K-距离较小
- 噪声点的K-距离较大
- 肘部是核心点和噪声点的分界

**代码实现**：
```python
def plot_k_distance(X, k=4):
    k_distances = []
    for i in range(len(X)):
        distances = sorted([np.linalg.norm(X[i] - X[j]) 
                           for j in range(len(X)) if i != j])
        k_distances.append(distances[k-1])
    
    k_distances.sort(reverse=True)
    plt.plot(k_distances)
    plt.xlabel('Points (sorted)')
    plt.ylabel(f'{k}-distance')
    plt.show()
```

</details>

### 练习4：算法对比分析 ⭐⭐⭐

假设你有一个包含月亮形状和两个圆形的数据集：

```
      🌙                    ● ● ●
     🌙  🌙                 ● ● ●
    🌙    🌙                  ● ● ●
     🌙  🌙
      🌙
```

**问题1**：K-Means能正确识别这3个簇吗？为什么？

<details>
<summary>参考答案</summary>

**K-Means的表现**：❌ 不能正确识别

**原因**：
1. K-Means假设簇是**球形的**，基于到中心的距离分配点
2. 月亮形状不是球形，K-Means会将其错误分割
3. K-Means倾向于产生大小相近的圆形簇
4. 月牙的凸包包含了圆形簇的部分区域，会产生混淆

**预期结果**：
- K-Means可能会将月牙横向或纵向切开
- 或者将月牙的一部分与圆形簇合并

</details>

**问题2**：DBSCAN能正确识别这3个簇吗？需要注意什么参数设置？

<details>
<summary>参考答案</summary>

**DBSCAN的表现**：✅ 能够正确识别

**为什么可以**：
1. DBSCAN基于**密度连接**，不假设簇的形状
2. 月牙虽然是弯曲的，但内部点的密度是连续的
3. 只要月牙上的点在ε范围内连成一片，就能被识别为一个簇

**参数设置要点**：
1. **ε设置**：
   - 必须小于月牙与圆形簇之间的最小距离
   - 必须大于月牙内部相邻点的最大距离
   
2. **MinPts设置**：
   - 对于二维数据，MinPts ≥ 4（2×维度数）
   - 需要小于月牙最窄处的点数

3. **具体建议**：
   - 先用K-距离图确定ε的大致范围
   - 从MinPts=4开始尝试
   - 如果簇被分割，增大ε；如果噪声太多，减小ε

</details>

**问题3**：层次聚类（Ward法）的表现会如何？

<details>
<summary>参考答案</summary>

**Ward法的表现**：⚠️ 部分可以，但不如DBSCAN

**Ward法的特点**：
1. 最小化合并后的**方差增量**
2. 倾向于产生**大小相近、形状紧凑**的簇
3. 对球形簇效果好，对非球形簇效果一般

**预期结果**：
- 两个圆形簇：应该能被正确识别 ✓
- 月牙形状：可能会被分割成多个小簇 ✗

**原因分析**：
- 月牙的不同部分距离较远，Ward法会认为它们是不同的簇
- Ward法关注"紧凑性"，而月牙是"延展性"的结构
- 单链接（Single Linkage）对月牙效果好，但Ward法不行

**改进建议**：
- 如果一定要用层次聚类，尝试**单链接（Single Linkage）**
- 但单链接对噪声敏感，需要先做噪声过滤

</details>

### 练习5：综合评估 ⭐⭐⭐

给定以下聚类结果（使用欧氏距离）：
- 簇1：{(0,0), (1,1), (2,0)}
- 簇2：{(10,10), (11,11)}

**问题1**：计算每个点的轮廓系数

<details>
<summary>参考答案</summary>

**第一步：计算点(0,0)的轮廓系数**

a(0,0) = 到同簇其他点的平均距离
       = (d((0,0),(1,1)) + d((0,0),(2,0))) / 2
       = (√2 + 2) / 2 ≈ 1.707

b(0,0) = 到最近其他簇的平均距离
       = 到簇2的平均距离
       = (d((0,0),(10,10)) + d((0,0),(11,11))) / 2
       = (√200 + √242) / 2 ≈ 14.79

s(0,0) = (b - a) / max(a, b) = (14.79 - 1.707) / 14.79 ≈ 0.884

**第二步：计算点(1,1)的轮廓系数**

a(1,1) = (d((1,1),(0,0)) + d((1,1),(2,0))) / 2
       = (√2 + √2) / 2 = √2 ≈ 1.414

b(1,1) = (d((1,1),(10,10)) + d((1,1),(11,11))) / 2
       = (√162 + √200) / 2 ≈ 13.43

s(1,1) = (13.43 - 1.414) / 13.43 ≈ 0.895

**第三步：计算点(2,0)的轮廓系数**

a(2,0) = (d((2,0),(0,0)) + d((2,0),(1,1))) / 2
       = (2 + √2) / 2 ≈ 1.707

b(2,0) = (d((2,0),(10,10)) + d((2,0),(11,11))) / 2
       = (√128 + √170) / 2 ≈ 12.18

s(2,0) = (12.18 - 1.707) / 12.18 ≈ 0.860

**第四步：计算簇2的点的轮廓系数**

对于(10,10)和(11,11)：
- a = d((10,10), (11,11)) = √2 ≈ 1.414
- b = 到簇1的平均距离 ≈ 13.43
- s ≈ 0.895

</details>

**问题2**：计算这个聚类的整体轮廓系数

<details>
<summary>参考答案</summary>

**整体轮廓系数** = 所有点轮廓系数的平均值

```
s(0,0) ≈ 0.884
s(1,1) ≈ 0.895
s(2,0) ≈ 0.860
s(10,10) ≈ 0.895
s(11,11) ≈ 0.895

整体轮廓系数 = (0.884 + 0.895 + 0.860 + 0.895 + 0.895) / 5
             ≈ 4.429 / 5
             ≈ 0.886
```

**结果分析**：
- 轮廓系数 ≈ 0.886，非常接近1
- 说明这是一个**非常优秀**的聚类结果
- 两个簇分离得很开，内部很紧密

</details>

**问题3**：计算Dunn Index（簇间距离用单链接）

<details>
<summary>参考答案</summary>

**Dunn Index公式**：
```
Dunn Index = min(簇间距离) / max(簇内直径)
```

**第一步：计算簇间距离（单链接）**

单链接距离 = 两个簇中最近的两个点之间的距离

簇1到簇2的最近距离：
- d((2,0), (10,10)) = √[(10-2)² + (10-0)²] = √[64 + 100] = √164 ≈ 12.81
- 其他点对的距离更大

所以：min(簇间距离) ≈ 12.81

**第二步：计算簇内直径**

簇内直径 = 簇内最远的两个点之间的距离

簇1的直径：
- d((0,0), (2,0)) = 2（最大）

簇2的直径：
- d((10,10), (11,11)) = √2 ≈ 1.414

max(簇内直径) = max(2, 1.414) = 2

**第三步：计算Dunn Index**

```
Dunn Index = 12.81 / 2 = 6.405
```

**结果分析**：
- Dunn Index = 6.405，是一个很大的值
- 说明簇之间分离得很好，簇内部很紧凑
- 通常Dunn Index > 1就认为聚类质量不错

</details>

### 练习6：Lance-Williams递推公式 ⭐⭐⭐

**问题**：证明Lance-Williams公式在Complete Linkage参数下等价于：
```
d(C∪D, X) = max(d(C,X), d(D,X))
```

<details>
<summary>参考答案</summary>

**Lance-Williams公式**：
```
d(C∪D, X) = αc·d(C,X) + αd·d(D,X) + β·d(C,D) + γ·|d(C,X) - d(D,X)|
```

**Complete Linkage的参数**：
```
αc = 0.5, αd = 0.5, β = 0, γ = 0.5
```

**代入公式**：
```
d(C∪D, X) = 0.5·d(C,X) + 0.5·d(D,X) + 0·d(C,D) + 0.5·|d(C,X) - d(D,X)|
          = 0.5·[d(C,X) + d(D,X) + |d(C,X) - d(D,X)|]
```

**分情况讨论**：

**情况1**：d(C,X) ≥ d(D,X)
```
|d(C,X) - d(D,X)| = d(C,X) - d(D,X)

d(C∪D, X) = 0.5·[d(C,X) + d(D,X) + d(C,X) - d(D,X)]
          = 0.5·[2·d(C,X)]
          = d(C,X)
          = max(d(C,X), d(D,X))
```

**情况2**：d(C,X) < d(D,X)
```
|d(C,X) - d(D,X)| = d(D,X) - d(C,X)

d(C∪D, X) = 0.5·[d(C,X) + d(D,X) + d(D,X) - d(C,X)]
          = 0.5·[2·d(D,X)]
          = d(D,X)
          = max(d(C,X), d(D,X))
```

**结论**：
在两种情况下，都有：
```
d(C∪D, X) = max(d(C,X), d(D,X))
```

这正是Complete Linkage的定义！证毕。

</details>

### 练习8：OPTICS算法研究 ⭐⭐⭐⭐

DBSCAN对参数的敏感性很高，当数据集中存在不同密度的簇时，单一的ε值无法同时识别所有簇。

**OPTICS算法**（Ordering Points To Identify the Clustering Structure）是DBSCAN的改进版。

**任务**：
1. 查阅资料，理解OPTICS算法的核心思想
2. 解释OPTICS中的"可达距离"（reachability distance）和"核心距离"（core distance）概念
3. 写出OPTICS和DBSCAN的主要区别
4. （选做）尝试用Python实现OPTICS算法的核心逻辑

<details>
<summary>参考答案框架</summary>

**核心思想**：
OPTICS不直接生成聚类结果，而是生成一个**排序**和**可达图**，从中可以提取任意密度层次的簇。

**核心距离（Core Distance）**：
```
core_dist(p) = 点p到其第MinPts个最近邻居的距离
             （如果|Nε(p)| < MinPts，则为UNDEFINED）
```

**可达距离（Reachability Distance）**：
```
reachability_dist(p, o) = max(core_dist(o), dist(o, p))
```

**与DBSCAN的主要区别**：
1. DBSCAN使用单一ε值，OPTICS可以提取多密度层次
2. OPTICS输出排序和可达图，需要后处理才能得到簇
3. OPTICS更适合发现密度差异大的簇

</details>

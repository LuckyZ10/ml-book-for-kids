## 39.1 从2D到3D：为什么需要神经辐射场？

### 39.1.1 一张照片不够

在第38章，我们学会了如何从噪声生成逼真的2D图像。但现实世界是三维的——当你走在街上，你可以从不同的角度看同一座建筑，看到它的正面、侧面、背面。

**传统方法的局限**：
- **3D建模软件**：需要专业艺术家手工创建，耗时耗力
- **摄影测量**：需要数百张照片，计算量巨大
- **深度相机**：只能捕捉几何，无法处理反光、透明材质

**问题**：能否只用几张普通照片，就"学会"整个3D场景，然后从新角度合成图像？

### 39.1.2 隐式表示 vs 显式表示

**显式3D表示**（传统方法）：
```
点云：存储数十万个(x,y,z)点
网格：存储顶点和面
体素：存储3D像素网格
```
**问题**：存储量大，分辨率受限，难以表示光滑表面

**隐式表示**（NeRF的创新）：
```
神经网络 F: (x,y,z,θ,φ) → (r,g,b,σ)
          位置和视角    →  颜色和密度
```

**优势**：
- **连续**：可以查询任意位置的值
- **紧凑**：一个神经网络存储整个场景
- **可微**：可以用梯度下降优化

### 39.1.3 费曼比喻：智能调色盘

想象一个超级智能的调色盘，你告诉它：
- "我在房间的角落"
- "面向窗户"

它就会告诉你那个位置和方向应该是什么颜色。

**NeRF就是这样的调色盘**——一个神经网络，学会根据位置和视角预测颜色和密度。

## 39.2 NeRF核心原理

### 39.2.1 5D辐射场

NeRF将场景表示为一个**5D函数**：

$$F(x, y, z, \theta, \phi) = (c, \sigma)$$

其中：
- **(x, y, z)**：3D空间坐标（位置）
- **(θ, φ)**：观察方向（方位角和极角）
- **c = (r, g, b)**：RGB颜色
- **σ**：体积密度（不透明度）

**为什么需要方向？**

同一点从不同角度看可能颜色不同（反光、镜面、半透明）：
- 看湖面：正面看是蓝色，侧面看是反射的天空
- 看窗帘：正面看透光，侧面看不透

### 39.2.2 体积渲染

**问题**：给定一条从相机出发的射线，如何计算该射线对应像素的颜色？

**费曼比喻：雾中的风景**

想象你站在浓雾中看一座山：
- 雾越浓的地方（密度高），你能看到山的机会就越小
- 体渲染就像是计算每缕雾气后面的风景有多少能穿透雾到达你的眼睛

**数学：体渲染方程**

对于射线上的点，我们累加颜色和密度：

$$\hat{C} = \sum_{i=1}^{N} T_i \cdot \alpha_i \cdot c_i$$

其中：
- **$T_i$**（透射率）：光线到达该点未被遮挡的概率
  $$T_i = \exp\left(-\sum_{j=0}^{i-1} \sigma_j \delta_j\right)$$
  
- **$\alpha_i$**（不透明度）：光线在该点被吸收的概率
  $$\alpha_i = 1 - \exp(-\sigma_i \delta_i)$$
  
- **$c_i$**：该点的颜色
- **$\delta_i$**：采样点间距

**直观理解**：
- 沿射线采样多个3D点
- 每个点贡献一些颜色，但会被前面的点遮挡
- 最终颜色是所有点贡献的加权平均

### 39.2.3 神经网络架构

NeRF使用两个MLP网络：

**网络1：密度预测**
```
输入：位置 γ(x) [63维]
输出：密度 σ + 特征向量
```

**网络2：颜色预测**
```
输入：特征向量 + 方向 γ(d) [27维]
输出：RGB颜色 c
```

**为什么分开？**
- 密度只与位置有关（与视角无关）
- 颜色与位置和视角都有关
- 这种设计减少了形状-光照歧义

### 39.2.4 位置编码

**问题**：MLP有频谱偏置，倾向于学习低频函数，难以捕捉高频细节（纹理、边缘）。

**解决方案**：位置编码（Positional Encoding）

将低维坐标映射到高维频率空间：

$$\gamma(p) = \left[p, \sin(2^0\pi p), \cos(2^0\pi p), \sin(2^1\pi p), \cos(2^1\pi p), \ldots, \sin(2^{L-1}\pi p), \cos(2^{L-1}\pi p)\right]$$

**费曼比喻：音阶与和弦**

想象你只听一个低音C，很难分辨旋律。但如果你加上C的高八度、再高八度，形成和弦，就能听出更多细节。

位置编码就像给坐标加上"高八度"，让神经网络能感知更精细的空间细节。

**参数**：
- 位置编码：L=10，输出63维
- 方向编码：L=4，输出27维

### 39.2.5 训练过程

**输入**：多张不同视角的照片 + 相机参数

**输出**：训练好的神经网络 F

**损失函数**（重建损失）：

$$\mathcal{L} = \sum_{r \in R} \| \hat{C}(r) - C(r) \|^2$$

其中：
- $R$：所有射线的集合
- $\hat{C}(r)$：渲染的颜色
- $C(r)$：真实照片的颜色

**训练流程**：
1. 随机采样一批射线（来自不同照片）
2. 沿每条射线采样3D点
3. 用MLP预测每个点的密度和颜色
4. 通过体渲染计算像素颜色
5. 计算与真实照片的误差
6. 反向传播，优化网络参数
7. 重复直到收敛

**采样策略**：
- **分层采样**：粗采样 + 细采样（在密度高的区域采样更多点）
- **重要性采样**：根据上一轮的密度自适应调整采样点

## 39.3 改进与扩展

### 39.3.1 Mip-NeRF：抗锯齿

**问题**：NeRF对远处物体或斜视角会产生锯齿（走样）。

**原因**：NeRF假设每条射线是无限细的光线，忽略了像素覆盖的区域。

**Mip-NeRF的解决方案**：
- 用**圆锥形视锥体**替代射线
- 对视锥体内的区域进行积分
- 引入**集成位置编码（IPE）**

**IPE公式**：

$$\gamma_{\text{IPE}}(\mathbf{x}, \Sigma) = \left[\sin(2^k\pi \mathbf{x}) \cdot \exp\left(-\frac{1}{2}(2^k\pi)^2 \text{diag}(\Sigma)\right), \cos(\ldots)\right]$$

**效果**：
- 自动适应不同尺度的细节
- 远处物体更平滑，近处更清晰
- 单模型支持多尺度

### 39.3.2 NeRF++：360°无界场景

**问题**：NeRF假设场景在单位球内，无法处理无界场景（如户外）。

**NeRF++的解决方案**：
- 将空间分为**近景**（单位球内）和**远景**（球外）
- 远景使用**倒球参数化**：
  $$\mathbf{x}' = \frac{\mathbf{x}}{\|\mathbf{x}\|^2} + \left(2 - \frac{1}{\|\mathbf{x}\|}\right) \cdot \frac{\mathbf{x}}{\|\mathbf{x}\|}$$
- 两个区域分别用不同的MLP处理

**效果**：可以建模360°无界场景（如山顶全景）。

### 39.3.3 Mip-NeRF 360：无界场景优化

**综合了Mip-NeRF和NeRF++的优点**：
- 圆锥采样 + IPE（抗锯齿）
- 倒球参数化（无界场景）
- 改进的采样策略

**效果**：目前新视角合成的最佳方法之一。

## 39.4 加速方法：从小时到秒

### 39.4.1 为什么NeRF慢？

**训练慢**：需要1-2天
- 每条射线采样数百个点
- 每个点需要两次MLP前向传播
- 需要数万次迭代才能收敛

**渲染慢**：实时性差
- 生成一帧需要数百万次MLP查询
- 无法达到实时渲染（30 FPS）

**目标**：实现实时渲染，让NeRF能在游戏、VR中应用。

### 39.4.2 Instant-NGP：哈希编码革命

**核心思想**：用哈希表存储特征，而非MLP计算。

**传统NeRF**：
```
位置 → [大MLP] → 颜色和密度
```

**Instant-NGP**：
```
位置 → [哈希表查询] → 特征向量 → [小MLP] → 颜色和密度
```

**多分辨率哈希编码**：

| 层级 | 分辨率 | 哈希表大小 |
|------|--------|-----------|
| 0 | 16³ | 2¹⁴ |
| 1 | 32³ | 2¹⁴ |
| ... | ... | ... |
| 15 | 524288³ | 2¹⁴ |

- **粗层级**：捕捉低频结构
- **细层级**：捕捉高频细节
- **哈希冲突**：通过梯度平均自动解决

**费曼比喻：图书馆索引卡**

想象一个巨型图书馆，每本书的位置由一个索引卡决定。Instant-NGP的多分辨率哈希编码就像是这样一套索引系统——从粗到细快速找到每本书（特征），而不需要遍历整个图书馆（大MLP）。

**效果**：
- **训练时间**：从1-2天 → **5秒-5分钟**
- **渲染速度**：60+ FPS（1080p）
- **质量**：与原始NeRF相当

### 39.4.3 3D Gaussian Splatting：实时渲染革命

2023年SIGGRAPH最佳论文，彻底改变神经渲染领域。

**核心思想**：不用神经网络，直接用**3D高斯点云**表示场景！

**什么是3D高斯？**

每个高斯由以下参数定义：
- **位置** μ ∈ ℝ³（中心）
- **协方差** Σ ∈ ℝ³ˣ³（形状和大小）
- **颜色** c ∈ ℝ³（RGB或球谐函数）
- **不透明度** α ∈ [0,1]

**高斯函数**：

$$G(\mathbf{x}) = \exp\left(-\frac{1}{2}(\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu})\right)$$

**费曼比喻：彩色泡泡**

想象数以万计的彩色肥皂泡漂浮在空中，每个泡泡中心最浓，边缘渐渐消失。当我们从某个角度看，这些泡泡叠加在一起就形成了一幅画面。

**3DGS就像是用软软的"电子泡泡"表示世界。**

**渲染：高斯溅射（Splatting）**

1. **投影**：将3D高斯投影到2D图像平面
   $$\Sigma' = \mathbf{J} \mathbf{W} \boldsymbol{\Sigma} \mathbf{W}^T \mathbf{J}^T$$
   （W：世界到相机变换，J：投影雅可比矩阵）

2. **排序**：按深度排序高斯

3. **混合**：从近到远混合高斯颜色
   $$C = \sum_{i} c_i \alpha_i \prod_{j=1}^{i-1}(1 - \alpha_j)$$

**优化策略**：

| 策略 | 目的 | 方法 |
|------|------|------|
| 自适应密度控制 | 细节不足时增加高斯 | 在梯度大的区域分裂或克隆高斯 |
| 周期性剪枝 | 移除无效高斯 | 删除透明度接近0的高斯 |
| 球谐函数 | 视角依赖颜色 | 用9个球谐系数表示颜色 |

**球谐函数简介**：

用一组基函数表示视角依赖的颜色：

$$c(\mathbf{v}) = \sum_{l=0}^{L-1} \sum_{m=-l}^{l} c_{lm} \cdot Y_l^m(\mathbf{v})$$

- v：观察方向（单位向量）
- $Y_l^m$：球谐基函数
- $c_{lm}$：可学习的系数

通常L=3，共9个系数，可以表示平滑的视角变化。

**效果**：
- **训练时间**：10-30分钟
- **渲染速度**：**100+ FPS**（1080p）
- **质量**：与Mip-NeRF 360相当
- **可编辑性**：可以直接操作高斯（移动、删除、添加）

### 39.4.4 方法对比

| 方法 | 表示 | 训练时间 | 渲染FPS | 可编辑性 |
|------|------|---------|---------|----------|
| NeRF | MLP | 1-2天 | <1 | ❌ |
| Mip-NeRF 360 | MLP | 48h | 0.07 | ❌ |
| Instant-NGP | 哈希+小MLP | 5秒-5分钟 | 60+ | ❌ |
| 3D Gaussian Splatting | 高斯点云 | 10-30分钟 | **100+** | ✅ |

**结论**：3DGS是目前最佳的速度-质量-可编辑性平衡点。

## 39.5 文本到3D生成

### 39.5.1 问题：从文字创造3D世界

给定文本描述：
- "一只穿着宇航服的猫"
- "一座飘浮在云中的城堡"

生成对应的3D模型，可以从任意角度查看。

**挑战**：
- 没有3D训练数据（3D数据稀缺）
- 需要多视角一致性（不同角度看到的应该一致）
- 需要符合文本描述

### 39.5.2 DreamFusion：开创之作

**核心思想**：利用预训练的2D扩散模型（如Imagen）来指导3D优化！

**Score Distillation Sampling (SDS)**：

**费曼比喻：雕塑家的批评家**

想象一个雕塑家（3D模型）在工作，旁边站着一个艺术评论家（扩散模型）。雕塑家从不同角度展示自己的作品，评论家根据文本描述给出建议："这里应该更像一只猫"。SDS就是评论家指导雕塑家改进的过程。

**数学**：

$$\nabla_\theta \mathcal{L}_{\text{SDS}} = \mathbb{E}_{t, \epsilon} \left[ (\epsilon_{\text{pred}} - \epsilon) \cdot \frac{\partial x}{\partial \theta} \right]$$

其中：
- $x$：从NeRF渲染的2D图像
- $\epsilon_{\text{pred}}$：扩散模型预测的噪声
- $\epsilon$：实际添加的噪声

**流程**：
1. 从随机NeRF开始
2. 随机采样相机视角，渲染2D图像
3. 用扩散模型评估图像与文本的匹配度
4. 计算SDS梯度，优化NeRF
5. 重复直到收敛

**效果**：首次实现文本到3D生成，但：
- 需要数小时优化
- 可能有Janus问题（多面怪）
- 几何质量有限

### 39.5.3 Magic3D：两阶段优化

**问题**：DreamFusion直接在NeRF上优化，分辨率受限。

**Magic3D解决方案**：

**第一阶段（低分辨率）**：
- 优化Instant-NGP（哈希编码）
- 快速获得粗略几何

**第二阶段（高分辨率）**：
- 提取网格（Marching Cubes）
- 优化可微分渲染器
- 生成高质量纹理

**效果**：
- 分辨率提升4倍
- 几何质量更好
- 速度提升2倍

### 39.5.4 MVDream：多视角一致性

**问题**：DreamFusion从单视角优化，容易出现"Janus问题"——物体在不同视角看起来不一致（如正面和背面都有脸）。

**MVDream解决方案**：
- 同时生成4个视角（前、右、后、左）
- 多视角扩散模型确保几何一致性
- 相机条件编码到扩散模型中

**效果**：
- 解决Janus问题
- 360°一致性更好
- 几何更准确

    """将输入x编码到高频空间"""
    encodings = [x]
    for l in range(L):
        encodings.append(torch.sin(2**l * np.pi * x))

**体渲染**：

```python
def raw2outputs(raw, z_vals, rays_d):
    """将原始网络输出转换为像素颜色"""
    # 分离密度和颜色
    raw_density = raw[..., 3]
    raw_color = raw[..., :3]
    
    # 计算不透明度：α = 1 - exp(-σδ)
    dists = z_vals[..., 1:] - z_vals[..., :-1]
    alpha = 1.0 - torch.exp(-F.relu(raw_density) * dists)
    
    # 计算透射率：T = cumprod(1 - α)
    transmittance = torch.cumprod(
        torch.cat([torch.ones_like(alpha[..., :1]), 1.0 - alpha + 1e-10], dim=-1),
        dim=-1
    )[..., :-1]
    
    # 加权求和
    weights = alpha * transmittance
    rgb = torch.sum(weights[..., None] * torch.sigmoid(raw_color), dim=-2)
    
    return rgb, weights
```

## 本章小结

### 核心概念回顾

| 概念 | 关键理解 |
|------|----------|
| **NeRF** | 用神经网络隐式表示5D辐射场 |
| **体积渲染** | 沿射线积分密度和颜色 |
| **位置编码** | 将坐标映射到高频空间，克服MLP频谱偏置 |
| **Mip-NeRF** | 圆锥采样+IPE，抗锯齿 |
| **Instant-NGP** | 哈希编码，训练从小时到秒 |
| **3DGS** | 高斯点云表示，实时渲染 |
| **SDS** | 从2D扩散模型蒸馏3D知识 |

### 关键公式

1. **体渲染**：$\hat{C} = \sum_{i} T_i \cdot \alpha_i \cdot c_i$
2. **位置编码**：$\gamma(p) = [\sin(2^k\pi p), \cos(2^k\pi p)]_{k=0}^{L-1}$
3. **SDS梯度**：$\nabla_\theta \mathcal{L}_{\text{SDS}} = (\epsilon_{\text{pred}} - \epsilon) \cdot \frac{\partial x}{\partial \theta}$

### 技术演进时间线

```
2020: NeRF诞生
2021: Mip-NeRF, PlenOctrees
2022: Mip-NeRF 360, Instant-NGP, DreamFusion
2023: 3D Gaussian Splatting, Magic3D, MVDream
2024+: 实时、可编辑、生成式成为主流
```

### 实践要点

- 从简单场景（如单个物体）开始
- 使用Instant-NGP快速原型验证
- 需要高质量相机标定
- 3DGS适合实时应用，NeRF适合高质量离线渲染
